Deprecated方法造成的问题

Feb 23, 2018

在平时的开发中，会遇到一些方法被划上横线的情况，点进去可以看到方法上有@Deprecated注释字样，标注为不建议使用的方法。一直以来认为这样方法的功能性都是好的，可能是更新之后提供了别的优化过的方法或是什么，但不导致正常使用。但是今天写了一个程序，根据关键字解析地址。在发送API请求时，在IDE测试都是好好的，发布到环境中运行就是出错，排查了半天发现是URL编码出了问题，程序中用了一个URLEncoder.encode(String s)方法，而且方法被划上了横线。此方法在将”测试”字符进行URL编码时，在IDE中结果为：”%E6%B5%8B%E8%AF%95”，到了发布环境中却成了”%B2%E2%CA%D4”，导致API响应的都是错误的数据。点进URLEncoder方法发现类中提供了一个新的方法URLEncoder.encode(String s, String enc)，enc参数为指定编码，这个方法没有@Deprecated修饰，测试如下：

在encode(String s)上的注释可以看到：

@deprecated The resulting string may vary depending on the platform’s default encoding. Instead, use the encode(String,String) method to specify the encoding.

所以encode(String s)并不知道字符串的编码，它读取了平台的默认编码，从而导致不同的平台产生了不同的结果。可见之后使用@Deprecated方法的时候都要小心了，可能一时半会功能正常，但是长久就会出现问题，还不容易被发觉，造成极大的安全隐患。
