用Java写一个爬虫

May 1, 2018

利用Quartz和线程池写了一个小框架，网络请求客户端使用HttpClient，解析html使用Jsoup和正则。

系统中的主要类有两个：scheduler包下的QuartzMaster和CrawlerJob，实现了任务的调度和抽象。

QuartzMaster里面有个runIt方法，把写好的任务模型和爬取线程的路径放进去就行了。task_count为每一次从库中获取的任务数量，thread_count为线程数量，cron是一个表示式，可以设置各种时间调度。

CrawlerJob实现了Quartz提供的Job接口，在execute方法中构建任务和线程模型。

这里根据任务模型的路径去相应的任务表里取出count个任务，再把任务集合和爬取线程的配置传给runTask方法。

根据线程数大小的指定，获取一个相应大小的线程池，这些数目的线程将被重用。将任务循环分配给这些线程执行，如果task_count为10，thread_count为3，那么前3个任务将各自获得一个线程执行，其中一个执行完之后闲置的线程将会接着去执行后面剩余的任务。

这里使用反射的主要原因是：Task任务的取出及线程的分配和创建都交给了框架，所以使用者不用实例化任何对象，只要告诉框架模型的位置即可，框架将去根据调度和传参动态加载。
