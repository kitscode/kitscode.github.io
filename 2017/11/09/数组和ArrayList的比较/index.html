数组和ArrayList的比较
Google时看到一篇文章，对数组和ArrayList访问数据的速度做了一个比较，这里记录下个人的测试和理解。原文链接:http://robaustin.wikidot.com/how-does-the-performance-of-arraylist-compare-to-array

之前看了些ArrayList的源码，知道了ArrayList内部维护了一个Object数组，实际上所有对ArrayList的操作都是基于数组，也就是说ArrayList只是一个普通的Java类，只不过对数组进一步进行了封装，提供给了用户一些好用的方法。所以在对数组在进行数据访问时，只是进行按索引取值:array[index],而ArrayList.get(index)则是封装了除了按索引取值外一些其他的东西，导致的速度的减慢。

进入ArrayList的源码，定位get(int index)方法，发现有2个，原来其中一个属于内部类SubList，这个类好像是可以产生ArrayList的部分视图。

另一个方法：
public E get(int index) {
rangeCheck(index);
return elementData(index);
}

发现ArrayList在索引数组前，还进行了一次索引是否越界的检查
private void rangeCheck(int index) {
if (index >= size)
throw new IndexOutOfBoundsException(outOfBoundsMsg(index));
}

在索引数组后还有一次泛型的转换
E elementData(int index) {
return (E) elementData[index];
}

加上不同方法间的调用，故花费了许多其他的时间，导致了效率的相对低下。但对于Java面向对象和封装的思想来说ArrayList还是加快了开发效率和使用便捷度，具体可根据使用时的不同需要进行选择。
